- ML-DSA-87 signatures: Not used in data path
- **Packet Overhead**: 40 bytes (visible in logs)

- Original: 132 bytes

**Why ShadowMesh is Faster than Tailscale**:

1. **Direct UDP Path**: ShadowMesh uses dedicated UDP sockets without additional NAT layers

4. **ChaCha20 Efficiency**: Highly optimized on ARM (Apple Silicon) and x86 (Intel Xeon)

**Trade-offs**:
- Higher CPU usage (2.6x) for quantum-safe encryption

## Observations and Limitations

### Known Issues

1. **No Heartbeat Mechanism**:
   - Peers time out after 5 minutes of inactivity
   - Requires manual restart to re-register
   - **Impact**: Tests must be completed within 5-minute window
   - **Fix**: Implement periodic re-registration (planned for v0.2.0)

2. **Manual Peer Connection**:
   - Requires `-connect <peer-id>` flag to initiate connection
   - No automatic peer discovery from DHT
   - **Impact**: Cannot dynamically discover and connect to new peers
   - **Fix**: Kademlia DHT implementation (planned for v0.2.0+)

3. **Routing Complexity**:
   - macOS requires `sudo route add -net 10.10.10.0/24 -interface utun8`
   - Route not automatically configured
   - **Impact**: Extra manual setup step
   - **Fix**: Automatic route configuration in client

4. **NAT Candidate Publishing**:
   - Warning: `failed to publish candidates: 404`
   - Endpoint not implemented in simple discovery server
   - **Impact**: NAT traversal limited to direct connections
   - **Fix**: Full discovery server with NAT candidate support

### Alpha Code Limitations

**Not Yet Implemented**:
- Automatic peer discovery
- DHT (Kademlia) for decentralized operation
- Heartbeat/keepalive mechanism
- NAT hole punching with candidates
- Multi-hop routing
- Traffic obfuscation
- Performance optimizations (SIMD, kernel bypass, etc.)

**Current Implementation**:
- ‚úÖ Post-quantum cryptography (ML-KEM-1024, ML-DSA-87)
- ‚úÖ ChaCha20-Poly1305 symmetric encryption
- ‚úÖ Layer 3 TUN device networking
- ‚úÖ UDP transport for data
